# knowledge/ — RAG 知识库

---

## 这个文件夹是干什么的？

用一句话理解：**这里放的是"酒店员工手册"——Agent 需要"背下来"的酒店静态信息**。

客人问"早餐几点""游泳池在哪""WiFi密码多少"，这些信息不需要 LLM 自己编，也不需要写死在代码里。放在这里的文本文件会被转成向量索引，Agent 在回答前先查一遍知识库。

---

## 工作原理（RAG = 检索增强生成）

```
客人问："早上几点有早餐？"
  ↓
① rag_retriever.py 从 Chroma 向量库中搜索"早餐"
  ↓
② 找到最相关的知识片段：
  "自助早餐：7:00 - 10:00，位于1楼云端餐厅，中西式均有，住店客人免费"
  ↓
③ 把这段知识拼到 LLM 的 system prompt 里
  ↓
④ LLM 基于这段知识回答："早餐是早上7点到10点，在一楼云端餐厅，住店客人免费哦～"
```

---

## placeholder_info.txt 内容分类

| 类别 | 包含信息 |
|------|---------|
| 酒店基本信息 | 名称、地址、前台电话 |
| 餐厅信息 | 早餐/午餐/晚餐时间、地点、送餐费 |
| 设施信息 | 游泳池、健身房、商务中心、停车场、洗衣房 |
| 客房设施 | WiFi、床型、浴室用品、电压 |
| 服务信息 | 入住/退房时间、叫醒、行李、加床、婴儿床 |
| 周边信息 | 地铁站、便利店、医院 |

---

## embedding 模型

使用的 embedding 模型是 `sentence-transformers/all-MiniLM-L6-v2`：
- 文件大小：约 90MB
- 向量维度：384
- 语言支持：英文为主，中文可用（效果可接受）
- 首次运行会自动下载，之后缓存到本地

---

## 未来扩展

- 如果需要更精准的中文检索，可以替换为 `shibing624/text2vec-base-chinese`
- 如果知识量大（几百条），建议拆成多个 .txt 文件按类别索引
- 知识更新只需修改 txt 文件，重启 Agent 后自动重建索引
